    <h2>Lab 1 - System Diagnostics</h2>
    <h3>Steps to Access Device Manager:</h3>
    <ul>
      <li>Press Windows + X or right-click on the Start menu.</li>
      <li>Click on Device Manager.</li>
      <li>You will see categories like Display Adapters, Processors, Network Adapters, and Sound devices.</li>
      <li>Expand each category to view specific device information.</li>
      <li>Right-click a device to update, disable, uninstall, or view properties.</li>
    </ul>

    <h3>Steps to Access Task Manager:</h3>
    <ul>
      <li>Press Ctrl + Shift + Esc or Ctrl + Alt + Del → Task Manager.</li>
      <li>Go to the Performance tab to view CPU, RAM, disk, GPU, and network usage.</li>
    </ul>

    <h3>Steps to Access Event Viewer:</h3>
    <ul>
      <li>Press Windows + R → Type <code>eventvwr.msc</code> → Press Enter.</li>
      <li>Expand Windows Logs: System, Application, and Security.</li>
      <li>Click entries to view their details and use filters to refine your search.</li>
    </ul>

    <h3>Questions Answered:</h3>
    <ul>
      <li><strong>CPU vs GPU:</strong> CPU handles general-purpose tasks, while GPU handles parallel and graphic-intensive tasks.</li>
      <li><strong>Internet vs Ethernet:</strong> Internet is the global network; Ethernet is the local wired connection.</li>
      <li><strong>Unix vs Linux:</strong> Unix is proprietary; Linux is open-source and Unix-like.</li>
      <li><strong>Unix Distribution:</strong> A complete OS built around the Unix/Linux kernel with user tools (e.g., Ubuntu, Fedora).</li>
    </ul>

    <h2>Lab 2 – Installing VirtualBox and Windows VM</h2>
    <h3>Installing VirtualBox:</h3>
    <ul>
      <li>Go to <a href="https://www.virtualbox.org">virtualbox.org</a>.</li>
      <li>Download for Windows and follow install prompts.</li>
    </ul>

    <h3>Installing a Windows VM:</h3>
    <ul>
      <li>Click New → Name VM → Choose Windows 10.</li>
      <li>Assign RAM and create a virtual hard disk (20GB+).</li>
      <li>Attach Windows ISO and start VM.</li>
      <li>Complete Windows setup as usual.</li>
    </ul>

    <h3>Adding Another Drive & File Systems:</h3>
    <ol>
      <li>In VM settings → Storage → Add new VDI hard disk (dynamically allocated, e.g. 10GB).</li>
      <li>Start Windows VM → Go to Disk Management → Initialize the new disk.</li>
      <li>Create a new partition, assign a drive letter, and format (e.g., NTFS).</li>
    </ol>

    <h4>File System Types:</h4>
    <ul>
      <li><strong>Windows:</strong> NTFS, FAT32, exFAT</li>
      <li><strong>Linux:</strong> ext4, XFS, Btrfs</li>
    </ul>

    <h2>Lab 5 – Apache Web Server and Samba File Sharing</h2>
    <h3>Apache Web Server:</h3>
    <ul>
      <li>Install Apache: <code>sudo apt install apache2</code></li>
      <li>Start: <code>sudo systemctl start apache2</code></li>
      <li>Test: Go to <code>http://localhost</code></li>
      <li>Default directory: <code>/var/www/html</code></li>
    </ul>

    <h3>Samba File Sharing:</h3>
    <ol>
      <li>Install Samba: <code>sudo apt install samba</code></li>
      <li>Create folder: <code>mkdir /home/user/shared</code></li>
      <li>Edit config file: <code>/etc/samba/smb.conf</code></li>
      <li>Add:
        <pre>
[SharedFolder]
path = /home/user/shared
read only = no
browsable = yes
        </pre>
      </li>
      <li>Set password: <code>sudo smbpasswd -a user</code></li>
      <li>Restart Samba: <code>sudo systemctl restart smbd</code></li>
    </ol>

    <h2>Lab 6 – Concurrency</h2>
    <p>Concurrency allows multiple tasks to progress at once, improving system responsiveness.</p>

    <h3>Real-life Example:</h3>
    <ul>
      <li>Multiple tabs open in a browser.</li>
      <li>Typing in Word while downloading a file.</li>
    </ul>

    <h3>Demo:</h3>
    <ul>
      <li>Open terminal and run: <code>ping google.com</code></li>
      <li>Open another terminal: <code>top</code></li>
      <li>Open Firefox and browse the web simultaneously.</li>
    </ul>

    <h3>Concept:</h3>
    <ul>
      <li><strong>Concurrency:</strong> Tasks appear simultaneous via context switching.</li>
      <li><strong>Parallelism:</strong> Tasks run truly simultaneously on multi-core CPUs.</li>
    </ul>
